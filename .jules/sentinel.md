## 2026-06-06 - Secure IDOR Prevention via Middleware\n**Vulnerability:** API routes were trusting the `userId` passed in the request body without verifying it against the actual authenticated user's token, leading to an Insecure Direct Object Reference (IDOR) vulnerability.\n**Learning:** Next.js Edge middleware often relies on raw JWT decoders, meaning the user ID might be stored as `.uid`, `.sub`, or `.user_id`. \n**Prevention:** Centralize authentication in `middleware.ts` by decoding the token and passing the verified user ID downstream via a secure `x-user-id` header. In the API route, explicitly check that the request body's `userId` matches the `x-user-id` header. Ensure compatibility by checking `uid`, `sub`, and `user_id` when decoding the token.
