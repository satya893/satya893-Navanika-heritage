## 2024-06-07 - IDOR in Checkout and Order Actions
**Vulnerability:** Insecure Direct Object Reference (IDOR) where users could manipulate `userId` in API payloads to perform actions (like place orders, clear cart, cancel orders) on behalf of other users. Additionally, relying on fail-open security checks like `if (authUserId && authUserId !== userId)` leaves the route vulnerable if the header is unexpectedly empty. Finally, taking the `x-user-id` header without explicitly deleting the client-provided header in middleware could allow header spoofing.
**Learning:** Next.js middleware is where token validation happens, but it doesn't automatically protect route logic from payload tampering. The `x-user-id` header must be sanitized first by `requestHeaders.delete('x-user-id')` before injection, and downstream API checks must be strict fail-closed (e.g., `if (!authUserId || authUserId !== userId)`).
**Prevention:** Always extract identity securely from verified tokens in middleware, inject it via headers (after stripping the client's version of the header), and use fail-closed authorization checks strictly comparing the token identity to the payload/document identity.
